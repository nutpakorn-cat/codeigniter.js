Codeigniter.js Works!

A quick guide to setup your application

Static file works! Credit: Pixabay/Noel_Bauza

This page is loaded by Welcome_controller.js inside application/controllers/ folder.

To get started, let's config the application ;D . Config files are inside /application/config/ There are two necessary files

  • config.js  —  server port, static file directory, session, body-parser (built-in), render engine
  • database.js  —  all about database

Server Port

First, let's take a look with config.js and change server port.

/**
 * Port
 * default: 8080
 */
exports.port = 8080;

Static file

Static files (css, js, images, fonts) are inside /assets folder. The files will appear on the url for an example you have /assets/abc.jpg, so you can get it from www.yoursite.com/abc.jpg

exports.staticDir = 'assets';

Session

To enable session, you have to set exports.sessionIsUse equal true and don't forget to set secret key! You can use whatever key that you want. for more details, visit: @expressjs/session

exports.sessionIsUse = true;

exports.session = {
  secret: 'DON'T FORGET TO SET THIS',
  cookie: {
    maxAge: 3600000
  },
  resave: false,
  saveUninitialized: true
};

BodyParser

Codeigniter.js uses built-in parser instead of body-parser, since Express 4.16 announced that they had bundled body-parser as express built-in middleware, but it's still the same as body-parser. for more details, visit: @expressjs/body-parser 

exports.expressParserIsUse = false;

exports.expressParser = {
  extended: false
};

Render Engine

There are 30 template engines that compatible with Codeigniter.js. You can disable it if you want to make API application. Default state is enable, and default render engine is Handlebars.

*** Don't forget to install your desired render engine before use
The template engines that have already installed are Handlebars and Jade.

exports.renderEngineIsUse = true;
exports.renderEngine = 'handlebars';

// Use .html extension
exports.renderExtension = 'html';
